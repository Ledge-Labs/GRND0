# Connecting Data and Capability Services

Connections are explicit grants. The runtime contains no private corpus and does not scan the host by default.

Local model serving is part of `core` and requires no cloud grant. Model
artifacts and lane wiring are defined in [Local inference](INFERENCE.md). The
`web` and `git` profiles add capability services; their grants remain separate
from profile activation.

## Document folder or Obsidian vault

`compose.knowledge.example.yml` mounts one operator-selected directory read-only at `/knowledge` and grants that container path:

```powershell
$env:GRND0_KNOWLEDGE_HOST_PATH = (Resolve-Path '.\example-knowledge').Path
docker compose -f docker-compose.yml -f compose.knowledge.example.yml up --build -d --wait
```

The disabled `operator-files` entry in `configs/knowledge-sources.example.json` becomes active after its `enabled` field is set to `true`. Markdown, text, reStructuredText, Python, JSON, YAML, and TOML files are eligible. Hidden repository data, editor state, dependency trees, oversized files, binary files, and paths outside the grant are excluded.

Direct folder retrieval is lexical and read-only. A matching turn records `operator-files`, a relative file locator, and a content digest in its receipt.

## Local knowledge index

The runtime can build a SQLite text index from the same read grant:

```powershell
$payload = @{ source_id='vault-index'; root='/knowledge' } | ConvertTo-Json
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8000/api/v1/knowledge/ingest -Headers $headers -ContentType application/json -Body $payload
```

Ingestion replaces documents for the named source atomically. Stored locators are relative to the granted root. The `local-index` connector becomes available when indexed documents exist. This mechanism provides a persistent local RAG surface without shipping knowledge contents.

## Vector store

The `VectorStoreSource` protocol in `services/orchestrator_daemon/app/connectors/base.py` defines the adapter boundary. The bundled HTTP adapter sends:

```json
{"query":"retrieval query","limit":6}
```

The configured service returns:

```json
{"hits":[{"id":"document-id","text":"bounded passage","score":0.82,"sha256":"optional digest"}]}
```

`operator-vector` is enabled in `configs/knowledge-sources.example.json` after `GRND0_VECTOR_QUERY_URL` and `GRND0_VECTOR_TOKEN` are set. The `full` profile includes Qdrant as infrastructure; collection creation and embedding selection remain operator configuration because they determine data governance and model behavior.

## Recursive retrieval

Retrieved evidence above `GRND0_RLM_THRESHOLD_CHARS` enters the named `recursive-context` harness in bounded chunks. The resulting working set retains source identity and is recorded as condensed retrieval. This RLM-style reduction prevents unbounded context stuffing while preserving the evidence chain.

## Web grounding

Web capability requires both the service profile and the runtime grant:

```dotenv
GRND0_ALLOW_WEB=1
```

```powershell
docker compose --profile web up --build -d --wait
```

The web-research service searches a public search origin, rejects non-public destinations, follows only validated redirect hops, and uses the bundled browser harness for rendered reading when available. Receipts label evidence as `operator-web` and retain public URLs. The default grant is off.

## Local git

Local repository capability requires the Gitea profile and runtime grant:

```dotenv
GRND0_ALLOW_GIT=1
```

```powershell
docker compose --profile git up -d --wait
```

The bootstrap service creates or rotates the local administrator with credentials generated by `tools/bootstrap-env.ps1`. Gitea binds to loopback. The connector reads text files from authenticated repositories and labels evidence as `operator-git`. Repository mutation is not performed by the retrieval connector.

## External tools

Client-supplied OpenAI function definitions and Anthropic tool definitions are forwarded to the synthesis model. Structured tool requests return to the calling client. The runtime does not claim execution until a tool result is supplied in a later compatible request.

Internal shared tool connections remain an empty declaration in `configs/mcp/connections.yaml`. Adding a connection requires an explicit operator grant and a corresponding adapter.

## Observable change

`GET /api/v1/capabilities` reports active sources and grants. `grnd0_receipt.retrieval` records whether a turn used model knowledge only, connected evidence, no matching connected evidence, or recursively condensed evidence. Connecting a matching folder, index, vector store, web service, or repository therefore changes both the evidence available to synthesis and the receipt provenance.

Model-serving grants are separate from knowledge grants. Native GPU and
WSL/Linux model servers connect through explicitly enabled local backends;
they do not grant access to files, vector stores, web content, or repositories.
The serving procedures and receipt fields are defined in
[INFERENCE.md](INFERENCE.md).
